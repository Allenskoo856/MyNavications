# 贡献指南

感谢您愿意为 MyNavications 做贡献！下面是贡献的流程和准则。

## 贡献流程

1. Fork 本仓库
2. 新建分支：`git checkout -b feature/your-feature`
3. 提交代码：遵循提交规范（参见 `docs/04-development-setup.md`）
4. 提交 PR，并在描述里说明变更目的和回归测试步骤

## 代码审核标准

- 通过单元测试（如果新增功能包含测试）
- 代码风格整洁、注释清晰
- 无敏感信息（如密码、密钥）提交
- 对数据库变更需提供迁移脚本或说明

## 行为准则

- 尊重他人，文明交流
- 开放透明，详细描述问题与解决方案
